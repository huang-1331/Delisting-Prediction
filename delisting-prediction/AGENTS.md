# Codex 작업 지침

## 프로젝트 목표

이 프로젝트의 목표는 KOSDAQ 제조업 상장기업의 재무제표 데이터를 기반으로 특정 시점 `T` 이후 12개월 이내 상장폐지 여부(`상장폐지_t12`)를 예측하는 모델을 개발하고, 그 성능을 Altman Z-Score와 비교하는 것이다.

최종 연구 질문은 다음과 같다.

> 재무제표 기반 머신러닝 모델은 기존 Altman Z-Score보다 상장폐지 예측 성능이 우수한가?

## 작업 순서

각 작업은 다음 순서를 따른다.

1. 목표 확인
2. 구현 계획 작성
3. 구현
4. 결과 검증
5. 문서 업데이트

불확실한 데이터 정의, 컬럼명, 외부 출처, 라벨링 기준은 임의로 확정하지 말고 `TODO`로 표시한다.

## 데이터 누수 금지

모든 모델 입력은 반드시 `T` 시점에 관측 가능한 데이터만 사용한다.

금지 사항:

- `T` 이후 재무제표 수치 사용
- `T` 이후 상장폐지 여부나 상장폐지 사유를 특성으로 사용
- 라벨 생성 과정에서 만든 미래 정보 컬럼을 모델 입력으로 포함
- 전체 기간 통계량을 사전에 계산해 train/test 분할 전 특성으로 사용하는 방식

예측 구조:

```text
T 시점 데이터
-> 모델 입력
-> T+12개월 상장폐지 여부
```

## 평가 원칙

Accuracy를 주요 지표로 사용하지 않는다. 클래스 불균형이 예상되므로 다음 순서로 모델을 평가한다.

1. Recall
2. F1 Score
3. PR-AUC
4. Precision
5. Accuracy

반드시 계산할 지표:

- Recall
- Precision
- F1 Score
- ROC-AUC
- PR-AUC
- Confusion Matrix

## 모델 개발 순서

모델은 반드시 다음 순서로 구현한다.

1. Altman Z-Score
2. Logistic Regression
3. Random Forest
4. XGBoost

Altman Z-Score를 가장 먼저 구현하고, 이후 머신러닝 모델의 성능 개선 여부를 비교한다.

## 코드 작성 규칙

- Python 코드는 `src/` 아래에 기능별 모듈로 작성한다.
- API 키와 비밀번호는 절대 하드코딩하지 않는다.
- DART API 키는 환경변수 `DART_API_KEY`에서 읽는다.
- 함수에는 목적, 입력, 출력이 드러나는 docstring을 작성한다.
- 데이터 스키마가 확정되지 않은 부분은 `TODO` 주석으로 남긴다.
- 모델 학습 코드는 재현 가능한 random seed를 사용한다.
- 공통 설정은 `src/config.py`에 둔다.
- 반복되는 평가 로직은 `src/evaluation.py`로 분리한다.

## ChatGPT 프로젝트 연결 지침

사용자가 언급한 ChatGPT의 `상장폐지 프로젝트` 프로젝트 내용이 제공되면, Codex는 해당 내용을 우선 참고하여 코드와 문서를 작성한다.

현재 로컬 Codex 작업 환경에서는 ChatGPT 프로젝트 공간에 직접 접근하는 기능이 자동으로 제공되는지 확인할 수 없다. 이 기능을 정상적으로 사용하려면 다음 중 하나가 필요하다.

- 사용자가 ChatGPT 프로젝트의 관련 내용을 이 대화에 제공
- 해당 프로젝트 자료를 이 저장소 또는 접근 가능한 파일로 저장
- ChatGPT 프로젝트 내용을 읽을 수 있는 별도 커넥터 또는 도구 제공

위 조건이 충족되지 않으면 Codex는 현재 저장소의 파일과 사용자가 제공한 지침만을 기준으로 작업한다.
